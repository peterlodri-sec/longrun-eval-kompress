The Voting Ensemble Paradox

Why adding more voters makes your ensemble worse — and how a 3× penalty fixes it. Interactive demo.

The Setup

You have N compression checkpoints voting on which tokens to keep. Each was trained on a different quality floor — some are great, some are bad. You'd think voting together would help. It doesn't.

What's happening?

Under k-of-N drop voting, the ensemble evicts a token if ≥k voters say "drop." Under asymmetric floors, each stratum has a different weakest voter. The ensemble's eviction decision on each stratum is the k-th order statistic of the voter indicators — the k-th weakest on that stratum.

  • k=1 (AND): collapses to the per-stratum weakest — any single drop vote vetoes the keep
  • k=N (OR): collapses to the per-stratum strongest — all must agree to drop
  • k=majority: collapses to the per-stratum median — the middle voter decides

The ensemble's global operating point is a convex combination of per-stratum k-th-weakest voters — a point occupied by no individual checkpoint and Pareto-dominated by any strong single voter. Adding more voters cannot help because each stratum stays pinned by its own k-th weakest.

The Fix: 3× Asymmetric Loss Modulation

The 3.0× heavy false-eviction penalty on critical-syntactic tokens (numbers, paths, error codes) shrinks each voter's false-eviction set fastest on its weakest strata. This lifts the collapse floor uniformly — the paradox still holds mechanically, but the floor it collapses to is no longer low.

L = L_base + 3.0 × L_crit
L_crit = false-eviction loss on must-keep tokens (Sapir-Whorf term)

Empirical confirmation

The 3-checkpoint majority ensemble (v3+v4+v5) scores 0.031 below the best single model. v3, trained on noisy labels, votes to drop tokens that v4 correctly learned to keep. Under majority voting, v3's drops combine with v5's stratum-specific regressions to overrule v4's correct keeps.

Key finding

Across 17 trained models, 8 teachers, 4 architectures, and $1.76 in compute: label quality is the bottleneck, not model capacity or data quantity. v15 (983 pairs, largest dataset) regressed to 0.878. v11 (larger encoder) collapsed to keep_rate 0.517. Only label-quality interventions (self-labeling, C3 distillation, higher λ) improved the heretic metric.
